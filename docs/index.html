Sepsis Vitals

Offline sepsis prediction for low-resource hospitals

Vitals-only sepsis prediction for clinics without ICU data infrastructure.

A gradient-boosted model trained on six basic vitals aims for AUROC >= 0.82 four hours before clinical recognition, with no labs, imaging, continuous monitoring, or cloud dependency.

Project Phases

Hard-coded research roadmap from dataset construction through open-source deployment, with gates designed to prevent the project from drifting into an ICU-only model.

Data · Months 2-6

Dataset construction

  • 3 years of partner-site vitals and outcomes
  • Sepsis-3 informed clinician adjudication
  • MIMIC filtered to six-vital subset for pre-training
  • Gate: >=8,000 labeled episodes
ML · Months 4-10

Model development

  • Logistic regression baseline
  • XGBoost / LightGBM primary model
  • qSOFA sensitivity +10pp target
  • Calibrated risk tiers and SHAP drivers
Validation · Months 9-15

Clinical validation

  • Prospective pilot at 2 hospitals
  • Silent mode before alert mode
  • Track antibiotic timing and mortality
  • Gate: alert fatigue below 20%
Deployment · Months 13-18

Scale & open-source

  • Docker container for Raspberry Pi 4
  • CSV, tablet, and monitor input modes
  • English, Swahili, French UI strings
  • MIT code, model card, implementation guide

Phase Detail

Each phase is written as an execution board: tasks, risks, gates, and outputs are visible without needing a backend.

Data Pipeline

Designed for prospective LMIC validation first, with public ICU datasets used only for tooling, pre-training, and stress testing.

01
Site extract

Paper abstraction, EMR export, OpenMRS table, CSV, tablet form, or monitor dump.

02
De-identify

Remove direct identifiers, date-shift where required, and keep site-owned linkages local.

03
Adjudicate

Clinicians review suspected infection, organ dysfunction, and ambiguous cases.

04
Engineer

Missingness, qSOFA, partial SIRS, shock index, deltas, rolling vitals.

05
Validate

Temporal and site holdout splits, calibration, fairness, and prospective pilot metrics.

Hard-Coded Study Gates

These gates are intentionally strict because the dataset, not the model architecture, is the project’s make-or-break risk.

AreaGateWhy it mattersFailure response
Dataset>=8,000 labeled episodes, <30% missing core vitalsEnough signal for site-aware validation.Extend abstraction or reduce scope to better-recorded departments.
Labels200 clinician review hours budgetedSepsis labels need clinical adjudication when labs are sparse.Use staged labels and sensitivity analysis.
ModelAUROC >= 0.82 and qSOFA sensitivity +10ppModel must beat deployable clinical baselines.Analyze missingness, horizon, and label noise before adding complexity.
ValidationProspective sensitivity >=0.78, alert fatigue <20%Clinical usefulness depends on workflow adoption.Tune thresholds by site and restart silent-mode monitoring.
DeploymentRuns offline on Raspberry Pi 4Cloud dependency breaks the low-resource premise.Reduce feature set, quantize, or ship simpler model fallback.

Evidence map

Research anchors translated into concrete product surfaces: burden framing, clinical action timing, pediatric scope, low-resource comparators, and alert fatigue risk.

WHO burden

Sepsis has global mortality weight and highest rates in LMICs, so the page now keeps burden, children, maternal risk, HIV/TB/malaria context, and infection prevention in the foreground.

WHO fact sheet
SSC 2026

The adult guideline update reinforces immediate treatment and resuscitation. The UI now separates prediction from clinical action and maps alerts to escalation workflow, not autonomous care.

SSC adult guidelines
NEWS2

NEWS2 uses bedside physiology, making it a serious comparator for a vitals-only model. The score lab now includes a NEWS2-style comparator alongside qSOFA and partial SIRS.

RCP NEWS2
UVA score

UVA was built from sub-Saharan African cohorts using accessible clinical features, so it is now treated as the local-context baseline to beat rather than an afterthought.

BMJ Global Health UVA
Phoenix pediatric criteria

Pediatric sepsis needs a separate pathway. The UI now flags pediatric scope as a dedicated research layer because Phoenix is not meant to replace early screening.

JAMA Phoenix criteria
Epic external validation

External validation and alert burden can break sepsis ML even in rich EHR settings. The validation monitor now shows alert-rate pressure next to AUROC and sensitivity.

JAMA Internal Medicine

Score comparator lab

Static but interactive scoring sandbox for explaining why the model must beat deployable clinical rules, not just produce a bigger AUROC.

Patient scenario

These scores are for research planning and UI prototyping only. Final thresholds must be calibrated on partner-site validation data.

Site readiness calculator

Quick planning tool for Phase 1: will a site produce enough plausible rows, and how much clinician review time does the labeling plan imply?

Prospective validation monitor

A dashboard sketch for the pilot: threshold tuning should show clinical sensitivity and alert fatigue together, or the model will be optimized for the wrong audience.
